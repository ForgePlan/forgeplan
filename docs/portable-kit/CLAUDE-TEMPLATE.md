## Workflow

### Перед каждой задачей:

```
1. Определи масштаб → Tactical / Standard / Deep
2. Если Standard+ → опиши в PRD перед кодом
3. Код + тесты одновременно
4. Подтверди результат (evidence)
5. Обнови прогресс в TODO.md
```

### При написании кода:

1. **Каждая новая публичная функция = тест сразу** — не переходи к следующей без теста
2. **После написания кода** — `{cargo test}` обязателен
3. **После значительных изменений** — аудит с 2+ агентами

## Git-конвенции

### Формат коммита:
```
тип(модуль): описание

Refs: PRD-001
```

### Типы: feat, fix, docs, test, refactor, chore, progress

### Ветки:
```
main                ← продакшн (теги)
dev                 ← интеграция
  feat/*            ← фичи (от dev, PR в dev)
  fix/*             ← фиксы
  release/v*        ← релизы (от dev, PR в main)
```

### Правила:
- **Merge commit, не squash** — squash теряет поздние коммиты
- **Не пушить после merge PR** — коммиты потеряются
- **Проверить после merge** — `git pull` и убедиться что изменения на месте
- **Не коммить в main/dev** — только через feature branch + PR
- **Не удалять ветки** — история решений
