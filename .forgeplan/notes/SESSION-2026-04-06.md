## Ключевые решения сессии

### Storage model (подтверждён ADR-003)
- Markdown в .forgeplan/ = source of truth (git-tracked)
- LanceDB = derived index (gitignored, rebuild: forgeplan scan-import)
- Industry pattern: как node_modules/, target/, .venv/
- Fresh clone: forgeplan init -y && forgeplan scan-import

### Git flow
- Branch protection rulesets: main + dev require PR
- Sync main→dev ВСЕГДА через PR (ruleset блокирует direct push)
- Pattern: git checkout -b sync/main-to-dev-vX.Y.Z && push && gh pr create --base dev
- Merge commit (не squash) для feature→dev и release→main
- НИКОГДА не мерджить PR пока все коммиты pushed и видны

### EPIC-002 v2.0 scope (FPF ADI analysis)
- H3 "Solo-first" победила (confidence 0.8)
- v2.0 = улучшить то что работает, не добавлять лишнее
- Tier 1 (must): Graph Intelligence, FPF Engine v2, Session command, Generate-docs, CI Linter
- Tier 2 (should): Built-in memory fallback, Diff command
- Tier 3 (defer): Multi-agent, TUI dashboard, Watch v2, VS Code, Nx
- Non-Goals: multi-agent orchestration, task management, replace Hindsight

### Memory architecture
- 22 альтернативы исследованы (Hindsight, Mem0, Ogham, mcp-memory-service, etc.)
- ForgePlan = structured memory (artifacts, R_eff, graph)
- External provider (Hindsight/Ogham) = unstructured chronicle (разговоры, мысли, хроника)
- Два слоя дополняют друг друга, не заменяют
- Top MCP-native: Ogham (97.2% recall), agent-memory-mcp (engineering-aware)
