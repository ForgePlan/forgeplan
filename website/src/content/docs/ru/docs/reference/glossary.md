---
title: Глоссарий
description: Ключевые термины, используемые в методологии Forgeplan
---

| Термин | Определение |
|------|-----------|
| **Артефакт** | Структурированный документ (PRD, RFC, ADR и т.д.), хранящийся в LanceDB |
| **R_eff** | Оценка надёжности = min(оценки_доказательств). Слабое звено, никогда не среднее значение |
| **Доказательство** | Результат теста, бенчмарк или вывод аудита, который поддерживает/ослабляет решение |
| **Уровень конгруэнтности (CL)** | Насколько контекст доказательства близок к контексту решения. CL3=идентичный, CL0=противоположный |
| **Глубина** | Уровень строгости: `Tactical`, `Standard`, `Deep`, `Critical` |
| **Конвейер** | Последовательность создания артефактов (например, PRD → RFC для стандартной глубины) |
| **ADI** | Цикл рассуждений: абдукция → дедукция → индукция |
| **Слепое пятно** | Активное решение без связанных доказательств |
| **Сирота (артефакт без связей)** | Артефакт без связей с другими артефактами |
| **Просроченный** | Артефакт, срок действия которого (`valid_until`) истёк. Оценка падает до 0.1 |
| **Замещённый** | Конечное состояние — заменён более новым артефактом |
| **Устаревший** | Конечное состояние — более не актуален |
| **Гейт валидации** | Проверка качества перед активацией артефакта (более 30 правил) |
| **Состязательная ревью** | Рецензент ОБЯЗАН найти проблемы. 0 найденных проблем = повторная ревью |
| **PRD** | Документ с требованиями к продукту — что и почему |
| **RFC** | Запрос комментариев — как строить |
| **ADR** | Запись архитектурного решения — почему именно такой подход |
| **Epic** | Группирует PRD, RFC, ADR в инициативу |
| **Spec** | Спецификация — контракты API, модели данных |
| **Проблема** | Сигнал с контекстом — ошибка, риск, наблюдение |
| **Решение** | Сравнение 2-3 вариантов с оценкой по принципу слабого звена |
| **Note** | Микро-решение, автоматически истекает через 90 дней |
| **Переоценка** | Переоценка просроченного артефакта |
| **Угасание доказательств** | Снижение оценки на основе TTL. Истёкший = 0.1 |
| **F-G-R** | Формальность, гранулярность, надёжность — измерения качества |
| **FPF** | First Principles Framework — методология рассуждений |
| **MCP** | Model Context Protocol — интеграция AI-агентов |
| **Роутинг** | Команда, которая предлагает глубину + конвейер для задачи |
| **Проекция** | Markdown-файл, сгенерированный из записи LanceDB |
| **Рабочее пространство** | Директория `.forgeplan/` с конфигурацией + хранилищем |
| **LanceDB** | Встраиваемая база данных для артефактов + векторных эмбеддингов |
