Показать компактный дашборд статуса проекта — общее количество артефактов, а также их количество, сгруппированное по типу и статусу. Это более легковесная альтернатива `forgeplan_health`, когда вам нужна только числовая сводка без анализа слепых пятен / сирот.

**Категория**: Дашборды и Графы

## Когда агент вызывает его

- **Быстрая проверка работоспособности** — сколько артефактов существует, сколько из них активных по сравнению с черновиками.
- **Отслеживание прогресса в спринтах** — сравнение количества с предыдущим базовым уровнем.
- **Ответ на вопрос "что находится в этом рабочем пространстве?"** без запуска полного сканирования состояния.
- **Быстрый путь** в случаях, когда `forgeplan_health` слишком ресурсоёмкий (например, для больших рабочих пространств).

Если вам нужны долги / риски / следующие действия, вместо этого вызовите `forgeplan_health`.

## Входные параметры

_Входные параметры отсутствуют. Вызовите этот инструмент с пустым объектом `{}`._

## Возвращает

```json
{
  "total": 184,
  "by_kind": {
    "prd": 42,
    "rfc": 18,
    "adr": 12,
    "epic": 5,
    "spec": 9,
    "evidence": 57,
    "problem": 34,
    "solution": 7,
    "note": 45,
    "refresh": 3
  },
  "by_status": {
    "draft": 12,
    "active": 98,
    "stale": 4,
    "superseded": 23,
    "deprecated": 47
  }
}
```

## Пример вызова

```json
{}
```

## Типичная последовательность

1. `forgeplan_status` — получить количество.
2. `forgeplan_list` с фильтром по `kind` / `status` — детализация по конкретному срезу.
3. `forgeplan_get` — прочитать конкретный артефакт.

## Эквивалент CLI

```bash
forgeplan status
```
