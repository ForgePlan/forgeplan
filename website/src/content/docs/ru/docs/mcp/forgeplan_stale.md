Обнаруживает артефакты, чья дата `valid_until` истекла. Такие артефакты являются "просроченными" — решение или доказательство, которое они содержат, может больше не соответствовать действительности и должно быть обновлено, продлено или отменено. Это первый шаг в цикле качества, связанном с устареванием доказательств.

**Категория**: Качество

## Когда агент вызывает этот инструмент

-   **Периодические проверки** — запускается еженедельно для выявления просроченных решений до того, как они незаметно снизят R_eff.
-   **Перед релизом** — подтверждение того, что ни один активный артефакт не выпускается с просроченными доказательствами.
-   **Начало сессии** — дополняет `forgeplan_health`, показывая срок действия для каждого элемента.
-   **После длительного отсутствия** — многие артефакты могли стать просроченными (`valid_until`) за время вашего отсутствия.

Просроченный ≠ недействительный — артефакт всё ещё существует, но его вклад в R_eff наказывается (CL0, вес 0.1 в формуле подсчёта баллов) до тех пор, пока он не будет продлён.

## Входные параметры

_Входные параметры отсутствуют. Вызовите этот инструмент с пустым объектом `{}`._

## Возвращает

```json
{
  "stale": [
    {
      "id": "EVID-047",
      "kind": "evidence",
      "valid_until": "2026-03-30",
      "days_expired": 12,
      "linked_to": ["PRD-039", "RFC-006"]
    },
    {
      "id": "ADR-003",
      "kind": "adr",
      "valid_until": "2026-04-01",
      "days_expired": 10
    }
  ],
  "total_stale": 2
}
```

## Типичная последовательность

1.  `forgeplan_stale` — список просроченных артефактов.
2.  Для каждого: принять решение о продлении, переоткрытии или отмене.
3.  `forgeplan_renew` для продления `valid_until` (если решение всё ещё актуально).
4.  `forgeplan_reopen` для создания нового черновика с отменой просроченного.
5.  `forgeplan_decay` — просмотр влияния на R_eff от оставшихся просроченных доказательств.

## Эквивалент в CLI

```bash
forgeplan stale
```
