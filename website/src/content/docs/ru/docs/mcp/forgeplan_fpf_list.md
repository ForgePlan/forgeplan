Возвращает полный каталог разделов базы знаний FPF — иерархическую таксономию, лежащую в основе механизма рассуждений Forgeplan. Каждый раздел представляет собой небольшую, самодостаточную часть First Principles Framework (цикл ADI, Trust Calculus, ограниченный контекст, explore/investigate/exploit и т.д.). `forgeplan_fpf_list` — это индекс агента: просматривайте его, решайте, какие разделы кажутся релевантными, затем получайте полное содержимое через `forgeplan_fpf_section` или ищите содержимое через `forgeplan_fpf_search`.

**Категория**: База знаний FPF

## Когда агент вызывает этот инструмент

- Первый вызов в сессии, когда агенту нужно вспомнить, какие темы FPF вообще существуют.
- Перед `forgeplan_fpf_section` — для подтверждения точного ID раздела (например, `B.3` для Trust Calculus).
- Когда пользователь спрашивает «есть ли в FPF что-либо о X?», и агент хочет просмотреть заголовки, прежде чем выполнять векторный поиск.
- В качестве проверки работоспособности после `forgeplan fpf ingest` — список должен возвращать около 204 разделов при свежем импорте.

## Что вы получаете в ответ

Плоский массив разделов, каждый из которых содержит:

| Поле | Тип | Описание |
|-------|------|-------------|
| `id` | `string` | ID раздела, например, `A.1`, `B.3`, `C.2.2`. |
| `title` | `string` | Читаемый человеком заголовок. |
| `summary` | `string` | Аннотация раздела в 1–2 предложения. |
| `chapter` | `string` | Родительская глава (`A`, `B`, `C`, …). |

Список упорядочен по ID, чтобы связанные разделы располагались рядом друг с другом.

## Входные параметры

_Входные параметры отсутствуют. Вызовите этот инструмент с пустым объектом `{}`._

## Возвращает

```json
{
  "total": 204,
  "sections": [
    { "id": "A.1", "chapter": "A", "title": "Why First Principles",
      "summary": "Мотивация для фреймворка; сравнение с эвристическим рассуждением." },
    { "id": "B.3", "chapter": "B", "title": "Trust Calculus",
      "summary": "Как доказательства (evidence) устаревают, уровни конгруэнтности (congruence levels), правило слабого звена (weakest-link) R_eff." },
    { "id": "C.2.2", "chapter": "C", "title": "Explore / Investigate / Exploit",
      "summary": "Три категории действий, привязанные к порогам R_eff." }
  ]
}
```

## Пример вызова

```json
{}
```

## Типичная последовательность

```
forgeplan_fpf_list
   ↓ выбрать интересные ID
forgeplan_fpf_section { id: "B.3" }    ← полное содержимое одного раздела
   или
forgeplan_fpf_search  { query: "…" }   ← поиск на уровне содержимого (по ключевым словам или BGE-M3)
```
