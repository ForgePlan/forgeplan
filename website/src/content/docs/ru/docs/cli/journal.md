Покажите **журнал решений** — хронологическую временную шкалу того, когда артефакты
были активированы, замещены, отменены или продлены, с аннотациями их
оценок R_eff. В отличие от [`log`](/docs/cli/log/), который записывает каждое изменение,
`journal` показывает только события решений — историю, которую вы рассказываете новому члену команды.

## Когда использовать

- Онбординг — "как мы сюда пришли?"
- Ретроспективы — какие решения хорошо себя показали, а какие были замещены
- Режим `--risk`: поиск активных решений без доказательств (слепых пятен) на
  временной оси
- Подготовка примечаний к выпуску — извлечение всего, что было активировано с момента тега

## Когда не использовать

- Вам нужны низкоуровневые CRUD-события → используйте [`forgeplan log`](/docs/cli/log/)
- Вам нужен текущий снимок → используйте [`forgeplan health`](/docs/cli/health/)

## Использование

```text
forgeplan journal [OPTIONS]
```

## Опции

```text
  -t, --type <TYPE>  Фильтровать по типу (adr, note, problem, solution)
      --risk         Показывать только рискованные решения (без доказательств, просроченные, с низким R_eff)
  -h, --help         Вывести справку
  -V, --version      Вывести версию
```

## Примеры

Полная временная шкала решений:

```bash
forgeplan journal
```

Только ADR — архитектурная запись, меньший набор:

```bash
forgeplan journal --type adr
```

Режим риска — активные решения без доказательств или с просроченными подтверждениями:

```bash
forgeplan journal --risk
```

## Интерпретация вывода

Одна строка на событие решения, новейшие первыми:

```
2026-04-11  ACTIVATE    PRD-046  R_eff=0.85   Docs v0.18.0 catch-up
2026-04-09  ACTIVATE    ADR-007  R_eff=1.00   Choose bm25 crate v2.3.2
2026-04-08  SUPERSEDE   PRD-018  → PRD-042    FPF KB vector search
2026-04-07  DEPRECATE   RFC-003  R_eff=0.10   (replaced by RFC-004)
```

| Столбец   | Значение                                                   |
|-----------|------------------------------------------------------------|
| Дата      | Дата события жизненного цикла                               |
| Событие   | ACTIVATE / SUPERSEDE / DEPRECATE / RENEW / REOPEN          |
| Артефакт  | ID решения                                                 |
| R_eff     | Эффективное доверие (`min(evidence_scores)`), 0.00–1.00    |
| Заголовок | Краткий заголовок                                          |

`R_eff = 0.00` для активного решения — это тревожный сигнал — это означает, что доказательства вообще не связаны. `--risk` выявляет именно такие.

## Как это работает

`journal` находится между `log` (сырые изменения) и `health` (текущий свод):

```
log         → journal    → health
всё           решения      текущее состояние
```

Используйте его, чтобы рассказать историю решений; используйте `health`, чтобы увидеть, где вы находитесь сейчас.
