`forgeplan status` выводит компактный снимок рабочего пространства: сколько артефактов каждого типа существует, каков их статус жизненного цикла и что изменилось недавно. Это команда "что есть в этом проекте?", а не "что сломано?" — для обнаружения проблем используйте `forgeplan health`.

Используйте её, когда открываете проект, к которому не прикасались неделю, когда хотите быстро оценить скорость спринта, или когда пишете отчёт о состоянии и вам нужны текущие показатели.

## Когда использовать

- Ориентация в незнакомом рабочем пространстве
- Написание еженедельного отчёта/отчёта о состоянии спринта
- Подтверждение, что `forgeplan scan-import` обнаружил недавно отредактированные файлы
- Проверка на адекватность после массового импорта или слияния
- Перед стендапом — посмотреть, что изменилось со вчерашнего дня
- Демонстрация — быстрый обзорный слайд для заинтересованных сторон

## Когда НЕ использовать

- Обнаружение технического долга или проблем с качеством — используйте `forgeplan health` (слепые пятна, сироты)
- Проверка отдельного артефакта — используйте `forgeplan get <ID>` или `forgeplan context <ID>`
- Гейтинг CI — status не имеет семантики кода выхода; используйте `health --ci`
- Поиск конкретного артефакта — используйте `forgeplan search` или `forgeplan list`

## Использование

```text
forgeplan status
```

## Опции

```text
  -h, --help     Вывести справку
  -V, --version  Вывести версию
```

### Пример 2: Проверка на адекватность после импорта

```bash
forgeplan scan-import
forgeplan status
```

Подтверждает, что ожидаемое количество артефактов было обнаружено после ручных правок markdown. Если количество выглядит неверным, повторно запустите `scan-import` с флагом `--verbose`.

### Пример 3: Однострочник для спринт-стендапа

```bash
forgeplan status | head -15
```

Вывести заголовок и количество в сообщение чата, без части с недавней активностью.

## Интерпретация вывода

- **Артефакты по типу** — общие суммы по каждому ArtifactKind, разделённые по статусу жизненного цикла. `draft`, `active`, `stale`, `superseded`, `deprecated`, `expired` — возможные состояния. Большое количество `draft` означает незавершённую работу; большое количество `superseded` означает здоровую эволюцию.
- **Недавняя активность** — события создания / обновления / связывания / активации / замещения за последние 7 дней. Извлекается из журнала решений.

Красные флаги:

- Много черновиков и нет активных — вы создаёте артефакты, но никогда их не завершаете
- Нулевая недавняя активность в течение нескольких дней — проект может быть заброшен или работа ведётся вне CLI (прямое редактирование markdown без `scan-import`)
- Количество доказательств непропорционально низкое по сравнению с количеством PRD — отсутствует покрытие доказательствами; запустите `forgeplan health` для получения подробностей
- Заметки с большим количеством `expired` записей — автоматическое истечение срока действия работает как задумано, никаких действий не требуется

## Как это вписывается в рабочий процесс

```
session start → [status] → health → route → Shape → ...
                   ^
             optional warmup
```

- **Status** отвечает на вопрос "что существует?"
- **Health** отвечает на вопрос "что сломано?"
- Запустите status для ориентации, затем health для списка действий
- Вместе они образуют пару дашбордов для начала сессии
