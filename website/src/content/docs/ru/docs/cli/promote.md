title: forgeplan promote
description: "Повышает статус сохранённой памяти до полноценного артефакта (PRD, RFC, ADR и т. д.)"
---

Превращает **сохранённую память** (созданную ранее с помощью `forgeplan remember`) в первоклассный артефакт — PRD, RFC, ADR, ProblemCard или Note. Используйте `promote`, когда наблюдение, которое вы записали несколько дней или недель назад, оказывается достаточно важным, чтобы заслужить структурированного обоснования, доказательства и жизненного цикла. Это мост между «Я что-то заметил» и «это решение, которое мы отслеживаем».

## Когда использовать

- Вы использовали `forgeplan remember` во время сессии, и наблюдение всё ещё актуально позже.
- Элемент памяти постоянно появляется в результатах поиска `forgeplan recall` — это сигнал, что он значимый.
- Вы хотите повысить статус неформального наблюдения до ProblemCard или PRD с надлежащими ссылками и оценкой.
- Постмортем: несколько воспоминаний из завершённого спринта заслуживают того, чтобы стать ADR для журнала решений.
- Уведомление задним числом: память, которая была тактической на тот момент, теперь накопила достаточно контекста, чтобы быть формализованной.

## Когда НЕ использовать

- Память эфемерна и станет просроченной через неделю — оставьте её как память, она автоматически исчезнет.
- Вы хотите создать артефакт с нуля — используйте [`forgeplan new`](/docs/cli/new/) или [`forgeplan generate`](/docs/cli/generate/).
- Вы фиксируете текущее решение прямо сейчас — используйте [`forgeplan capture`](/docs/cli/capture/).
- Память не содержит достаточно контента для заполнения разделов MUST — сначала дополните её контекстом или начните с [`forgeplan generate`](/docs/cli/generate/).

## Использование

```text
forgeplan promote --kind <KIND> <MEMORY_ID>
```

## Аргументы

```text
  <MEMORY_ID>  ID памяти для повышения статуса (например, mem-042, mem-auth-decisions)
```

## Опции

```text
      --kind <KIND>  Целевой тип артефакта: prd, rfc, adr, note, problem, epic, spec
  -h, --help         Вывести справку
  -V, --version      Вывести версию
```

Сначала запустите `forgeplan recall`, чтобы перечислить существующие памяти и скопировать правильный ID. Повышение статуса создаёт новый артефакт, используя тело памяти в качестве начального содержимого — вам всё ещё нужно заполнить все отсутствующие разделы MUST и запустить `validate` перед `activate`.

## Примеры

### Пример 1: Повышение статуса наблюдения до ProblemCard

```bash
forgeplan recall "search stale"
# → mem-042  "Users report search returns stale results after rename"
forgeplan promote mem-042 --kind problem
```

Создаёт PROB-NNN, заполненный содержимым памяти в качестве Signal и Context. Откройте полученный ProblemCard, добавьте Goals и индикаторы Anti-Goodhart, свяжите любые связанные Evidence и активируйте.

### Пример 2: Повышение статуса памяти до PRD

```bash
forgeplan promote mem-auth-decisions --kind prd
```

Превращает текущую память об обсуждении аутентификации в надлежащий PRD. Команда заполняет то, что может (заголовок, черновое описание проблемы), и оставляет остальное в виде заполнителей шаблона — относитесь к этому как к выводу [`forgeplan new`](/docs/cli/new/): заполните оставшиеся разделы MUST, затем `validate`.

### Пример 3: Повышение статуса памяти о решении до ADR

```bash
forgeplan promote mem-017 --kind adr
```

Полезно, когда память фиксирует обоснование решения, которое заслуживает включения в постоянный журнал решений. Полученный черновик ADR наследует тело памяти; просмотрите, добавьте Consequences / Alternatives и запустите поток validate → activate.

## Как это вписывается в рабочий процесс

```
(earlier session)  forgeplan remember  →  mem-NNN
                                              │
(later session)  forgeplan recall  →  promote --kind <kind>  →  fill MUST  →  validate  →  activate
```

`promote` — это **отложенный путь в фазу Shape**: наблюдение было дёшево записать (`remember`), но теперь оно заслуживает полного жизненного цикла. После повышения статуса новый артефакт входит в стандартный поток `validate → reason → code → evidence → review → activate` — ничего особенного, просто с начальным содержимым.
