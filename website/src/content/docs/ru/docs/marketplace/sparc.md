## Что такое SPARC?

SPARC — это структурированная методология разработки с 5 фазами, каждая из которых обрабатывается специализированным агентом. Доступен в качестве плагина на ForgePlan Marketplace.

## 5 фаз

### 1. Спецификация
Чётко определите проблему. Каковы требования, ограничения и критерии успеха?

**Агент:** `@specification`

### 2. Псевдокод
Разработайте логику решения до написания реального кода. Сосредоточьтесь на алгоритмах и потоках данных.

**Агент:** `@pseudocode`

### 3. Архитектура
Определите структуру системы — компоненты, интерфейсы, зависимости, модели данных.

**Агент:** `@architecture`

### 4. Уточнение
Реализуйте, тестируйте и итерируйте. Применяйте TDD, обрабатывайте граничные случаи, оптимизируйте.

**Агент:** `@refinement`

### 5. Завершение
Окончательный обзор, документация, подготовка к развёртыванию.

**Агент:** `@sparc-orchestrator` (координирует все фазы)

## Установка

```bash
npx skills add ForgePlan/marketplace --plugin agents-sparc
```

## Интеграция с Forgeplan

Фазы SPARC естественным образом соотносятся с конвейером Forgeplan:

| Фаза SPARC | Эквивалент Forgeplan |
|-------------|---------------------|
| Спецификация | PRD (Shape) |
| Псевдокод | RFC (Shape) |
| Архитектура | ADR (Validate + Reason) |
| Уточнение | Code (Build) |
| Завершение | Evidence + Activate (Prove) |

Используйте агентов SPARC для творческой работы, Forgeplan — для структурированной записи.
