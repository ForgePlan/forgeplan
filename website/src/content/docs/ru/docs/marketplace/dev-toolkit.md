## Что это делает

Плагин **dev-toolkit** предоставляет независимые от языка средства разработки для Claude Code и совместимых ИИ-агентов. Он охватывает полный цикл разработки: исследование темы, планирование спринта, создание реализации, аудит результата и восстановление контекста при возвращении.

## Установка

```bash
npx skills add ForgePlan/marketplace --plugin dev-toolkit
```

## Команды

| Команда | Описание |
|---------|-------------|
| `/audit` | Мультиэкспертный аудит кода (4 параллельных агента) |
| `/sprint "goal"` | Адаптивное планирование спринтов с волновым выполнением |
| `/recall` | Восстановление контекста сессии из памяти Hindsight |
| `/research "topic"` | Быстрое исследование с агентами Explore |
| `/deep-research "topic"` | Глубокое мультиагентное исследование (4-7 агентов), пишет отчёты |
| `/build "research-dir"` | Реализация на основе существующих отчётов об исследованиях |
| `/synthesize "dir1" "dir2"` | Объединение нескольких отчётов об исследованиях в единый план |
| `/do "task"` | Универсальный исполнитель задач — определяет правильный подход |
| `/wave "description"` | Быстрое волновое выполнение из текущего контекста |
| `/write-doc "type" "topic"` | Создание структурированных документов (RFC, руководство, отчёт, ADR) |
| `/team-up` | Запуск команд агентов для параллельной работы в нескольких областях |

## /audit -- Мультиэкспертный аудит

Запускает 4 агента параллельно, каждый из которых сфокусирован на своём аспекте качества:

1.  **Логика** — корректность, граничные случаи, состояния гонки, ошибки «на единицу»
2.  **Архитектура** — нарушения SOLID, связность, DRY, соглашения об именовании
3.  **Безопасность** — OWASP Top 10, векторы инъекций, пробелы в аутентификации/авторизации
4.  **Тесты** — пробелы в покрытии, качество тестов, отсутствие тестов для граничных случаев

Каждый агент сообщает о найденных проблемах с указанием уровня серьёзности: CRITICAL / HIGH / MEDIUM / LOW.

```
/audit
```

Лучше всего использовать перед созданием pull request. Используйте в паре с `/forge-audit` из плагина [forgeplan-workflow](/docs/marketplace/forgeplan-workflow/) для аудита с учётом методологии.

## /sprint -- Адаптивный спринт

Масштабируется от простой задачи до полноценного волнового спринта:

-   **1 задача** — просто выполняет её напрямую
-   **3-5 задач** — параллельное выполнение
-   **10+ задач** — волновой спринт с отслеживанием зависимостей, отчётами о прогрессе

```
/sprint "migrate database to PostgreSQL 16"
```

Планировщик спринта разбивает цель на упорядоченные волны, назначает приоритеты и отслеживает выполнение по волнам.

## /recall -- Восстановление контекста

Восстанавливает контекст сессии из памяти Hindsight — над чем вы работали, что было решено, что находится в ожидании.

```
/recall
```

Используйте в начале новой сессии, чтобы продолжить с того места, где вы остановились.

## /research -- Быстрое исследование

Изучает тему с помощью агентов Explore. Полезно для понимания принципов работы чего-либо перед реализацией.

```
/research "how does LanceDB handle schema evolution"
```

## /deep-research -- Мультиагентное исследование

Глубокое исследование с 4-7 параллельными агентами. Записывает структурированные отчёты в `docs/research/`. Используйте перед крупными архитектурными работами.

```
/deep-research "vector database comparison for embedding storage"
```

## /build -- Реализация на основе исследования

Берёт существующие отчёты об исследованиях, создаёт план реализации, а затем выполняет её.

```
/build "docs/research/vector-db-comparison"
```

## /synthesize -- Объединение исследований

Объединяет несколько отчётов об исследованиях в единый унифицированный план. Полезно, когда вы исследовали несколько связанных тем и нуждаетесь в единой согласованной дорожной карте.

```
/synthesize "docs/research/auth-options" "docs/research/session-management"
```

## /do -- Универсальный исполнитель

Принимает любое описание задачи и определяет правильный подход: исследование, сборка, аудит или их комбинация.

```
/do "add rate limiting to the API gateway"
```

## /wave -- Быстрое волновое выполнение

Планирует и выполняет из текущего контекста чата без отдельной фазы исследования. Хорошо подходит для хорошо понятных задач.

```
/wave "refactor the scoring module into separate files"
```

## /write-doc -- Генератор документов

Создаёт структурированные документы, используя шаблоны и контекст проекта. Поддерживает форматы RFC, руководства, отчёта и ADR.

```
/write-doc rfc "LanceDB migration strategy"
```

## /team-up -- Команды агентов

Запускает параллельные команды агентов для задач, охватывающих несколько областей (например, бэкенд + фронтенд + тесты). Каждый агент работает в своей области независимо, затем результаты объединяются.

```
/team-up
```

## Когда использовать

-   **Проекты Brownfield** — `/recall` для восстановления контекста, `/audit` для понимания качества
-   **Крупные спринты** — `/sprint` с волновым планированием
-   **Перед важными решениями** — `/deep-research`, затем `/build`
-   **Координация команды** — `/team-up` для параллельной работы в нескольких областях
-   **Быстрые задачи** — `/do` или `/wave` для хорошо понятных изменений
