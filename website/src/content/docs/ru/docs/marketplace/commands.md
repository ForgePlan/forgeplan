## Основные команды (forgeplan-workflow)

### /forge "описание задачи"
**Плагин**: forgeplan-workflow

Запускает полный цикл методологии Forgeplan: Route → Shape → Validate → Build → Evidence → Activate.

```
/forge "Добавить аутентификацию OAuth2"
```

Команда автоматически определяет глубину и создает нужные артефакты.

### /forge-cycle PRD-XXX
**Плагин**: forgeplan-workflow

Явный пошаговый цикл Forge с 8 фазами:

```
Phase 0: OBSERVE   → forgeplan health
Phase 1: ROUTE     → определить глубину
Phase 2: SPRINT    → спланировать волны
Phase 3: BUILD     → реализовать
Phase 4: AUDIT     → состязательная ревью
Phase 5: FIXES     → исправить HIGH/CRITICAL
Phase 6: EVIDENCE  → создать + оценить
Phase 7: COMMIT    → git + PR
```

### /forge-audit
**Плагин**: forgeplan-workflow

Многоэкспертный аудит кода с интеграцией методологии Forgeplan.

## Команды для разработки (dev-toolkit)

### /audit
**Плагин**: dev-toolkit

Запускает 4 параллельных экспертных агента:
- **Logic** — корректность, граничные случаи, состояния гонки
- **Architecture** — SOLID, связность, DRY, именование
- **Security** — OWASP Top 10, инъекции, аутентификация
- **Tests** — пробелы в покрытии, качество тестов

Сообщает о найденных проблемах с уровнем серьезности: CRITICAL / HIGH / MEDIUM / LOW.

### /sprint "цель"
**Плагин**: dev-toolkit

Адаптивное планирование спринта, которое масштабируется:
- 1 задача → просто выполнить
- 3-5 задач → параллельное выполнение
- 10+ задач → волновой спринт с отслеживанием зависимостей

### /recall
**Плагин**: dev-toolkit

Восстанавливает контекст сессии из памяти Hindsight — над чем вы работали, что было решено, что находится в ожидании.

## Команды FPF (плагин fpf)

### /fpf "вопрос"
**Плагин**: fpf

Автоматически выбирает правильный режим рассуждения на основе вашего вопроса.

### /fpf decompose "система"
Разбивает систему на ограниченные контексты с ролями, интерфейсами и обязанностями.

```
/fpf decompose наша система аутентификации
```

### /fpf evaluate "опции"
Сравнивает альтернативы с оценкой F-G-R и анализом доказательств.

```
/fpf evaluate React vs Vue vs Svelte для нашего SPA
```

### /fpf reason "проблема"
Структурированное рассуждение в цикле ADI: 3+ гипотезы → предсказания → проверка доказательств.

```
/fpf reason почему время ответа нашего API ухудшилось
```

## Команды Orchestra (плагин forgeplan-orchestra)

### /session
**Плагин**: forgeplan-orchestra

Полное восстановление контекста: память Hindsight + состояние Forgeplan + задачи Orchestra.

### /sync
**Плагин**: forgeplan-orchestra

Двунаправленная синхронизация между артефактами Forgeplan и задачами Orchestra.

## Команды UX (плагин laws-of-ux)

### /ux-review
**Плагин**: laws-of-ux

Проверяет фронтенд-код на соответствие 30 законам психологии UX. Сообщает о нарушениях с уровнем серьезности и предложениями по исправлению.

### /ux-law "название закона"
Ищет конкретный закон UX с примерами, нарушениями и лучшими практиками.

## Команды исследования и сборки (dev-toolkit)

### /research "тема"
**Плагин**: dev-toolkit

Быстрое исследование — изучение темы, поиск закономерностей, понимание принципов работы. Использует агентов Explore.

### /deep-research "тема"
**Плагин**: dev-toolkit

Глубокое многоагентное исследование (4-7 агентов). Записывает отчеты в `docs/research/`. Используйте перед началом серьезной работы.

### /build "research-dir"
**Плагин**: dev-toolkit

Реализация на основе существующих отчетов об исследованиях. Читает результаты исследования и создает план реализации.

### /synthesize "dir1" "dir2"
**Плагин**: dev-toolkit

Объединяет несколько отчетов об исследованиях в единый план. Полезно, когда вы исследовали несколько тем и нужен один план действий.

### /do "задача"
**Плагин**: dev-toolkit

Универсальный исполнитель задач — принимает любое описание и определяет правильный подход.

### /wave "описание"
**Плагин**: dev-toolkit

Быстрое волновое выполнение из текущего контекста чата. Без фазы исследования — только планирование и выполнение.

### /write-doc "тип" "тема"
**Плагин**: dev-toolkit

Создает структурированные документы: RFC, руководство, отчет, ADR. Использует шаблоны и контекст проекта.

### /team-up
**Плагин**: dev-toolkit

Запускает команды агентов для параллельной реализации. Лучше всего подходит для задач, охватывающих несколько областей (бэкенд + фронтенд).

## Проактивные предложения

Claude Code может предлагать команды на основе того, что вы делаете:

| Вы говорите | Предлагаемая команда |
|---------|-------------------|
| "изучи", "разберись", "найди" | `/research` |
| "глубоко изучи", "ресерч" | `/deep-research` |
| "напиши RFC", "доку" | `/write-doc` |
| Complex multi-step task | `/do` or `/sprint` |
| "ревью", "проверь" | `/audit` |
| Backend + frontend work | `/team-up` |
| Has research reports | `/build` |
| "объедини", "roadmap" | `/synthesize` |

## Когда что использовать

| Ситуация | Команда |
|-----------|---------|
| Запуск новой функции | `/forge "description"` |
| Проверка кода перед PR | `/audit` |
| Сложная многоэтапная задача | `/sprint "goal"` |
| Архитектурное решение | `/fpf evaluate "A vs B"` |
| Понимание системы | `/fpf decompose "system"` |
| Отладка проблемы | `/fpf reason "why X"` |
| Восстановление контекста | `/recall` or `/session` |
| Качество фронтенда | `/ux-review` |

## Цепочки рабочих процессов

```
Research → Build:
  /fpf decompose → /forge "implement X"

Multi-step feature:
  /forge "feature" → /audit → fix → /sprint "remaining tasks"

Architecture decision:
  /fpf evaluate "options" → /forge "implement winner"
```
