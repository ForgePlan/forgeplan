## Что такое FPF?

First Principles Framework (FPF) — это «Операционная система для мышления» — трансдисциплинарная архитектура для рассуждений. Она превращает необработанный интеллект (человеческий или машинный) в пригодные для использования в организации рассуждения.

FPF встроен в Forgeplan через команду `/fpf` и CLI `forgeplan fpf`.

## Три режима рассуждения

### /fpf decompose — Декомпозиция

Когда у вас есть сложная система и вам нужно понять её части.

```
/fpf decompose our authentication system
```

Вывод:
- Таблица: Контекст | Ответственность | Ключевые роли | Интерфейсы
- Диаграмма Mermaid с границами и связями
- Проверка на ошибку категории (роль против функции?)

**Когда использовать:** Начало новой функции, понимание существующей системы, планирование архитектуры.

### /fpf evaluate — Сравнение вариантов

Когда вам нужно выбрать между альтернативами с оценкой доказательств.

```
/fpf evaluate React vs Vue vs Svelte for our SPA
```

Вывод:
- Сильные стороны, слабые стороны, доказательства, отсутствующие доказательства для каждого варианта
- Баллы F-G-R: Формальность (0-3), Гранулярность (0-3), Надёжность (0-3)
- Матрица решений с рекомендацией
- Цикл ADI: гипотезы → предсказания → проверка доказательств

**Когда использовать:** Выбор технологии, архитектурные решения, анализ компромиссов.

### /fpf reason — Продумать

Когда вам нужно понять, почему что-то произошло, или как подойти к проблеме.

```
/fpf reason why our API response times degraded
```

Вывод:
- **Абдукция**: 3+ гипотезы
- **Дедукция**: Проверяемые предсказания для каждой гипотезы
- **Индукция**: Проверка по имеющимся доказательствам
- Оценённые гипотезы: подтверждены / ослаблены / опровергнуты
- Заключение с оставшимися неопределённостями

**Когда использовать:** Отладка, реагирование на инциденты, анализ проблем, обоснование архитектуры.

## База знаний FPF

Forgeplan включает полную спецификацию FPF в виде доступной для поиска базы знаний:

```bash
# Загрузить разделы спецификации FPF
forgeplan fpf ingest

# Поиск концепций
forgeplan fpf search "bounded context"

# Использовать контекст FPF в рассуждениях
forgeplan reason PRD-001 --fpf
```

## Ключевые концепции FPF

| Концепция | Что это значит |
|---------|--------------|
| **Ограниченный контекст** | Часть системы с чёткими границами и собственным словарём |
| **Trust Calculus** | Оценка уверенности: насколько вы можете доверять утверждению? |
| **F-G-R** | Формальность × Гранулярность × Надёжность — измерения качества |
| **Цикл ADI** | Абдукция → Дедукция → Индукция — структурированный цикл рассуждений |
| **Ошибка категории** | Путаница роли и функции, метода и работы |
| **Гамма-алгебра** | Как части объединяются в целое, сохраняя свойства |

## Установка

```bash
# Как плагин Claude Code
npx skills add ForgePlan/marketplace --plugin fpf

# Или используйте встроенный CLI
forgeplan fpf search "trust"
```

Полный каталог плагинов и инструкции по установке см. в [Обзоре Marketplace](/docs/marketplace/overview/).

:::tip
FPF наиболее эффективен в сочетании с системой артефактов Forgeplan. Используйте `/fpf decompose` для планирования, затем `forgeplan new prd` для сохранения результата в виде структурированного артефакта.
:::
